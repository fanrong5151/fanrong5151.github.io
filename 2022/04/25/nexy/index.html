数据库原理

1.数据库系统概述

1.1 数据与信息

  • 数据的定义，数据用来记录信息的可识别的符号组合，是信息的具体表现形式
  • 信息的定义，信息是人脑对现实世界事物的存在方式，运动状态以及事物之间联系的抽象反映
  • 信息 = 数据 + 语义

1.2 数据处理与数据管理

  • 数据（原料）→ 数据处理 → 信息（输出结果）
  • 数据管理：主要包括对数据的分类，组织，编码，存储，维护，检索等操作

1.3 数据库系统的组成

  • 主要由数据库，数据库用户，计算机硬件系统和计算机软件系统等几部分组成。
  • 数据库（DB）是存储在计算机内，有组织的，可共享的数据和数据对象（如表，视图，存储过程和触发器）的集合，这种集合按一定的数据模型组织，描述长期存储，同时能以安全可靠的方法进行数据的检索和存储。
  • 数据库的两个特点，集成性和共享性
  • 用户，对数据库进行存储，维护和检索等操作的人员。主要分为三类，最终用户，应用程序员和数据库管理员
  • 硬件系统，存储运行数据库的硬件设备

1.4 数据库系统内部体系结构

1.4.1 三级模式结构

  • 模式（Schema）是数据库中全体数据的逻辑结构和特征的描述，反映数据的结构，一个数据库只有一个模式，是数据在逻辑上的视图。
  • 外模式，又称为子模式，三级结构最外层，是数据库用户能看到并允许使用的那部分数据的逻辑和特征的描述，使用户的数据视图，用户视图。
  • 内模式，又称存储模式或物理模式，是三级结构中的最内层，是对数据库存储结构的描述，数据在数据库内部的表达方式，记录以什么方式存储。
  • 外模式可由多个，而内模式，模式只能各有一个。

1.4.2 二级映像与数据独立性

  • 为了能实现三个模式之间的联系和转换，提供了二级映像功能，分别是外模式/模式映像，模式/内模式映像。
  • 外模式/模式映像，模式描述全局逻辑结构，外模式描述数据局部逻辑结构。该映像确定了数据局部逻辑结构与全体结构之间的对应关系，保证数据与程序间的逻辑独立性。
  • 模式/内模式映像，是唯一的，他确定了数据的全局逻辑结构与存储结构之间的对应关系。保证数据物理独立性。
  • 优点，保证数据独立性，简化用户接口，有利于数据共享，数据安全。

1.5 数据库外部体系结构

  • 单用户结构，桌面数据库系统，单用户使用，不能共享
  • 主从式结构，大型主机带多终端的多用户结构系统，缺点是所有任务由主机完成，对主机性能要求较高。
  • 分布式结构，数据在逻辑上是整体，在物理上是分布的，不集中存储在一台服务器上。
  • 客户机/服务器结构，DBMS和数据库存放于数据库服务器上，应用程序和相关开发工具存放于客户机。
  • 浏览器/服务器结构，又称B/S结构，客户机和数据库服务器之间有一个中介应用服务器，将应用程序转移到应用服务器上。

1.6 数据库管理系统（DBMS）

1.6.1 DBMS 的主要功能

  • 数据定义功能，数据定义语言（Data Define Language，DDL），定义三级模式和二级映像，定义有关约束条件。
  • 数据操作功能，数据操纵语言（Data Manipulation Language，DML）实现对数据库的基本操作，检索，更新，修改，删除等。
  • 数据库运行管理功能，保证数据正确有效和数据库系统的正常运行。
  • 数据库建立和维护功能
  • 数据通信接口

1.6.2 DBMS组成

  • 语言编译程序，1）DDL编译程序，把各级源模式编译成各级目标模式。2）DML编译程序，将应用程序中的DML语句转换成可执行程序，实现对数据库的增删改查。
  • 系统运行控制程序，1）系统总程序，控制和协调个程序的活动，是核心程序；2）安全性控制程序；3）完整性控制程序，检查完整性约束条件，确保进入数据库中的数据正确性，有效性和相容性。
  • 并发控制程序，协调多用户，多任务环境下各应用程序对数据库的并发操作，保证数据一致性。
  • 数据存取和更新程序：实现对数据的增删改查。
  • 通信控制程序：实现用户程序与DBMS间的通信。
  • 还有文件读写，维护程序，缓冲区管理程序，存取路径管理程序，事务管理程序，运行日志管理程序等。
  • 数据字典（Data Ditionary，DD）用来描述数据库中有关信息的数据目录，包括数据库的三模式，数据类型，用户名和用户权限等有关数据库系统的信息，骑着系统状态目录表的作用，帮助用户，系统本身使用和管理数据库。

1.6.3 DBMS数据存取过程

  1. 用户使用特定数据操纵语言DML向DBMS发出请求
  2. DBMS接收请求并将该请求解释转换成机器代码指令
  3. DBMS依次检查外模式，外模式/模式映像，模式，模式/内模式映像及存储结构定义
  4. DBMS对存储数据库执行必要的存取操作
  5. 接受结果
  6. 对结果进行处理，如格式转换
  7. 将处理结果返回给用户
    上述过程包括安全性控制，完整性控制，以确保数据正确性，有效性和一致性。

1.7数据模型

1.7.1 分类

  • 概念模型，也称信息模型，对现实世界事物及其联系的第一级抽象。
  • 逻辑模型，也称数据模型，主要用于DBMS的实现。成熟地应用在数据库系统中的逻辑模型主要包括层次模型，网状模型，关系模型和面向对象模型。
  • 物理模型，是对数据最底层的抽象，描述数据在磁盘或磁带上的存储方式。

1.7.2 数据模型组成要素

数据模型通常由数据结构、数据操作和数据的完整性约束三个要素组成

1.8 其他概念

  • 实体，客观存在的事物
  • 属性，实体的某一特性
  • 实体型，用实体名及其属性名集合来抽象和描述共同类实体
  • 实体集，同型实体的集合成为实体集
  • 码，在实体型中能唯一标志一个实体店属性或属性集，称为实体的码
  • 域，某一属性的取值范围
  • 联系，属性之间的联系，实体型之间的联系

1.9 四种数据模型

  • 层次模型，树形数据结构来表示各类实体间的联系，层次模型的数据操纵主要有查询、插入、删除和修改，进行插入、删除和修改操作时要满足层次模型的完整性约束条件，进行删除操作时，如果删除双亲结点值，则相应的子女结点值也被同时删除
  • 网状模型，网状模型是采用有向图结构表示记录型与记录型之间联系的数据模型，网状模型的数据操纵主要包括查询、插入、删除和修改数据。进行插入、删除和修改操作时要满足网状模型的完整性约束条件。插入数据时，允许插入尚未确定双亲结点值的子女结点值，删除数据时，允许只删除双亲结点值。
  • 关系模型，关系模型的数据结构是一张规范化的二维表，它由表名、表头和表体三部分构成。
  • 面向对象模型
